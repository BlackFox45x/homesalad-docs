Development Roadmap

This section outlines the development plan for HomeSalad, including project phases, milestone targets, feature prioritization, and release strategy.

Development Philosophy

HomeSalad's development approach is guided by several core principles:

  • Systems-First Development: Building core gameplay systems before content creation to ensure solid foundations
  • Iterative Design: Regular playtesting and refinement cycles to improve gameplay feel and balance
  • Vertical Slice Strategy: Creating complete, polished sections before expanding to full scope
  • Community Feedback: Incorporating player input throughout development while maintaining vision
  • Sustainable Pace: Maintaining team health and avoiding crunch through realistic scheduling

Development Phases

The project is structured into six major development phases:

Phase 1: Pre-Production (3 Months)

Focus: Establishing design foundations, technical prototyping, and team alignment

Key Deliverables:

  • Complete game design document and story bible
  • Technical architecture plan and framework selection
  • Art style guides and audio direction documents
  • Core gameplay prototypes for gardening and building systems
  • Project management infrastructure and development roadmap

Milestones:

  • Design documentation approval
  • Technical prototype demonstration
  • Art direction approval
  • Team onboarding completion

Phase 2: Core Systems Development (6 Months)

Focus: Building the fundamental systems that power gameplay

Key Deliverables:

  • Character controller and animation system
  • Environmental transformation framework
  • Plant growth and gardening mechanics
  • Building and construction system
  • Basic NPC interaction framework
  • Time and season cycle implementation
  • Inventory and resource management

Milestones:

  • Playable systems prototype
  • First internal playtest
  • Technical systems review

Phase 3: Vertical Slice (4 Months)

Focus: Creating a polished, representative section of the game

Key Deliverables:

  • One complete district (Downtown Commons)
  • Core gameplay loop implementation
  • UI system and basic menus
  • Initial quest implementation
  • Character customization basics
  • Preliminary audio implementation
  • Visual effects for key interactions

Milestones:

  • Vertical slice completion
  • Focus group testing
  • Publisher/stakeholder presentation

Phase 4: Content Production (8 Months)

Focus: Building out the full game world and content

Key Deliverables:

  • All five districts with unique characteristics
  • Complete NPC roster and relationship systems
  • Full quest implementation for main storyline
  • Faction quests and reputation system
  • Complete plant and building libraries
  • Environmental states for all areas
  • Seasonal variations and weather system

Milestones:

  • Alpha release (feature complete)
  • Content completion review
  • Closed alpha testing

Phase 5: Polish and Optimization (3 Months)

Focus: Refining gameplay, fixing issues, and optimizing performance

Key Deliverables:

  • Bug fixing and quality assurance
  • Performance optimization for target platforms
  • UI refinement and accessibility implementation
  • Tutorial and onboarding improvements
  • Balance adjustments based on testing
  • Final audio mix and implementation
  • Localization integration

Milestones:

  • Beta release
  • Open beta testing
  • Performance benchmark targets met

Phase 6: Launch and Post-Launch (Ongoing)

Focus: Releasing the game and supporting it with updates and community engagement

Key Deliverables:

  • Release build preparation
  • Marketing and community support
  • Day one patch
  • Post-launch content updates
  • Community feedback implementation
  • Seasonal events and activities
  • Multiplayer implementation (post-launch)

Milestones:

  • Gold master
  • Public release
  • Post-launch update schedule

Feature Prioritization

Features are categorized based on their importance to the core experience:

Priority Description Example Features
P0: Core Essential features without which the game cannot function Character movement, environmental transformation, gardening system, building system
P1: Critical Features necessary for the complete gameplay experience NPC relationships, quest system, seasonal cycle, resource management
P2: Important Features that significantly enhance the experience Character customization, faction reputation, community projects, weather system
P3: Desirable Features that add depth but could be scaled back if needed Detailed skill system, cooking mechanics, decorative options, festival events
P4: Nice-to-Have Features that could be added post-launch if necessary Photo mode, advanced customization, additional districts, pet companions

Risk Management

Identified development risks and mitigation strategies:

Risk Impact Likelihood Mitigation Strategy
Environmental transformation system complexity High Medium Early prototyping, staged implementation, fallback to simpler version if needed
Performance issues with plant rendering Medium High LOD system, instancing, optimization sprints, scalable quality settings
Scope creep in content production High High Clear feature prioritization, modular design, regular scope reviews
Balancing complexity in gameplay systems Medium Medium Regular playtesting, data-driven design, configurable difficulty
Integration challenges with uMMORPG framework High Medium Technical spike early in development, dedicated integration team, fallback plans

Release Strategy

HomeSalad will follow a staged release approach:

Early Access (Year 1 Content)

  • Scope: Downtown Commons and Riverside Heights districts, core gameplay systems, Year 1 storyline
  • Duration: 6-8 months
  • Purpose: Gather player feedback, refine systems, build community
  • Update Cadence: Major updates every 4-6 weeks
  • Pricing: Reduced price with free upgrades to full version

Full Release (Complete Game)

  • Scope: All five districts, complete three-year storyline, all core features
  • Platforms: PC (Windows, Mac, Linux) initially, console ports later
  • Marketing: Focus on sustainability themes, community building, and unique gameplay
  • Launch Window: Avoid major competitive releases, target spring season

Post-Launch Support

  • Free Updates: Seasonal events, quality of life improvements, bug fixes
  • Expansion Content: New districts, storylines, and gameplay systems as paid DLC
  • Community Engagement: Regular dev streams, feature voting, community challenges
  • Multiplayer Implementation: Cooperative play added as major post-launch feature

Team Structure

The development team is organized into cross-functional groups:

Team Responsibilities Size
Core Systems Character controller, environmental systems, building mechanics, plant growth 5-6 developers
Content Creation World building, quest design, dialogue writing, NPC implementation 4-5 designers
Art & Audio Visual assets, animation, sound design, music composition 6-8 artists
UI/UX Interface design, player experience, accessibility, tutorials 2-3 specialists
QA & Community Testing, bug tracking, community management, player feedback 3-4 team members

Development Tools

Primary tools and technologies used in development:

  • Engine: Unity with uMMORPG Remastered framework
  • Version Control: Git with branching strategy for features and releases
  • Project Management: Jira for task tracking, Confluence for documentation
  • Art Pipeline: Blender for 3D modeling, Substance for texturing, Photoshop for 2D assets
  • Audio Production: FMOD for implementation, Pro Tools for composition and sound design
  • Testing: Automated test framework for core systems, playtesting protocols
  • Build System: Jenkins for CI/CD, automated build distribution

Design Note: The development roadmap for HomeSalad emphasizes building solid foundations before expanding content, with regular opportunities for testing and refinement. The staged release approach allows for community input while maintaining the core vision for the game.
