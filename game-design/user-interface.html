User Interface

This section details the user interface design for HomeSalad, including UI philosophy, HUD elements, menus, controls, and information visualization systems.

UI Design Philosophy

HomeSalad's user interface is designed around several core principles:

  • Environmental Integration: UI elements that blend with and complement the game world rather than obscuring it
  • Contextual Information: Providing relevant information based on current activities and player focus
  • Visual Clarity: Clear, readable elements with consistent visual language and intuitive iconography
  • Progressive Disclosure: Revealing complexity as needed rather than overwhelming players with options
  • Customization: Allowing players to adjust UI elements to suit their preferences and needs

The UI aesthetic draws inspiration from natural systems, sustainable design, and community bulletin boards, using organic shapes, earthy colors, and handcrafted textures to reinforce the game's themes.

HUD (Heads-Up Display)

HomeSalad features a minimalist HUD that provides essential information while maintaining immersion in the game world.

Core HUD Elements

Element Description Position Behavior
Status Indicators Player energy, time of day, season, weather conditions Top right corner Always visible, subtle design that expands on hover
Contextual Actions Available interactions with nearby objects or characters Bottom center Appears when interaction is possible, fades when not needed
Quick Access Bar Equipped tools, frequently used items, quick actions Bottom of screen Customizable slots, visual feedback on selection
Minimap Local area map showing nearby points of interest Top right corner Toggleable, can be expanded to larger view
Objective Tracker Current quest objectives and progress Top left corner Shows prioritized quest, can be expanded to show more

Contextual HUD Elements

Additional HUD elements appear based on current activities:

  • Gardening Overlay: Plant information, growth stages, soil conditions when gardening
  • Building Grid: Placement guides, resource requirements when constructing
  • Social Indicators: Relationship status, conversation options during interactions
  • Environmental Metrics: Biodiversity, water quality, soil health in relevant contexts
  • System Monitors: Energy flow, water management, resource cycles for created systems

HUD Customization

Players can customize their HUD through several options:

  • Opacity Settings: Adjust transparency of HUD elements
  • Scale Options: Change the size of UI components
  • Position Adjustments: Move certain elements to preferred locations
  • Information Density: Toggle between minimal and detailed information displays
  • Color Schemes: Select from accessibility-focused color options

Menu Systems

HomeSalad features several menu systems for different aspects of gameplay:

Main Menu

The entry point to the game with options for:

  • Continue Game (loads most recent save)
  • New Game (starts character creation)
  • Load Game (access to save files)
  • Multiplayer (host or join options)
  • Settings (game options and customization)
  • Credits (acknowledgments and team information)
  • Exit Game (returns to desktop)

In-Game Menu (Pause)

Accessed during gameplay with options for:

  • Resume Game (returns to gameplay)
  • Save Game (creates save file)
  • Load Game (access to save files)
  • Settings (game options and customization)
  • Help (tutorials and guidance)
  • Return to Main Menu (exits current game)

Player Journal

A multi-tabbed interface that serves as the main information hub:

Tab Content Functionality
Quests Active, available, and completed quests Track progress, set priorities, review outcomes
Map Full world map with discovered locations Fast travel, location information, custom markers
Skills Character skills and knowledge Track progression, review techniques, plan development
Relationships NPC and faction relationships View standings, gift preferences, conversation history
Calendar Seasonal events, schedules, deadlines Plan activities, track time-sensitive events
Collections Discovered plants, recipes, building designs Review collected knowledge, track completion
Notes Player-created notes and observations Record ideas, create to-do lists, save information

Inventory System

Manages player items and resources:

  • Carried Items: Personal inventory with weight/space limitations
  • Storage Systems: Home storage, community storage, specialized containers
  • Categorization: Items sorted by type, function, and source
  • Item Information: Detailed descriptions, properties, and usage options
  • Resource Management: Tracking of bulk materials and their quantities

Building Interface

Used for construction and placement activities:

  • Blueprint Selection: Categories of available structures and objects
  • Resource Requirements: Materials needed for construction
  • Placement Grid: Visual guides for positioning and orientation
  • Preview System: See how items will look before finalizing
  • Modification Tools: Options to adjust, move, or remove existing elements

Community Board

Central hub for community information and activities:

  • Event Calendar: Upcoming community events and activities
  • Project Tracking: Status of ongoing community initiatives
  • Resource Exchange: Requests and offers for materials and assistance
  • Voting System: Community decisions and policy proposals
  • Announcements: Important information and updates

Controller Support

Full controller support with mappings for:

  • Movement: Left analog stick
  • Camera: Right analog stick
  • Actions: Face buttons (A, B, X, Y or equivalent)
  • Menu Access: D-pad directions for quick access
  • Tool Selection: Shoulder buttons and triggers
  • Special Functions: Button combinations for advanced actions

Touch Controls

Mobile-optimized interface with:

  • Virtual Joystick: Movement control
  • Swipe Gestures: Camera control and menu navigation
  • Context-Sensitive Buttons: Actions based on current situation
  • Multi-Touch Support: Pinch to zoom, two-finger rotation
  • Tap Interactions: Direct object selection and manipulation

Information Visualization

HomeSalad uses several visualization systems to communicate complex information:

Environmental Indicators

  • Soil Health Visualization: Color gradients showing soil quality and composition
  • Water Flow Indicators: Animated lines showing water movement and drainage
  • Biodiversity Highlights: Visual indicators of species presence and interactions
  • Pollution Representation: Visual effects showing contamination levels and spread
  • Growth Potential: Overlays showing suitability for different plants and systems

Relationship Visualization

  • Social Network Map: Interactive diagram of character relationships
  • Faction Standing Meters: Visual representation of reputation with groups
  • Relationship Dimensions: Multi-axis display of trust, respect, and affinity
  • Influence Flows: Visualization of how relationships affect community dynamics
  • Gift Impact Preview: Indication of how actions will affect relationships

System Diagrams

  • Resource Cycles: Flowcharts showing material and energy movement
  • Ecological Webs: Visualizations of species interactions and dependencies
  • Community Networks: Maps of social connections and resource sharing
  • Seasonal Patterns: Circular diagrams showing yearly cycles and events
  • Progress Tracking: Visual representations of transformation over time

Feedback Systems

HomeSalad uses multiple channels to provide feedback to players:

Visual Feedback

  • Interaction Highlights: Subtle outlines around interactive objects
  • Action Effects: Visual representations of player actions and their impacts
  • Status Changes: Color shifts and animations showing state transitions
  • Progress Indicators: Fill bars, growing elements, and completion markers
  • Notification Icons: Symbols indicating new information or opportunities

Audio Feedback

  • Interaction Sounds: Distinct audio for different actions and materials
  • Success/Failure Cues: Tones indicating outcome of actions
  • Ambient Changes: Shifting soundscapes reflecting environmental health
  • UI Navigation Sounds: Subtle audio cues for menu interaction
  • Notification Alerts: Distinctive sounds for different types of updates

Haptic Feedback

  • Tool Use: Vibration patterns matching different activities
  • Environmental Conditions: Subtle feedback reflecting terrain and weather
  • System States: Pulses indicating resource flows and system functions
  • Notification Emphasis: Haptic alerts for important information
  • Contextual Resistance: Variable feedback based on materials and conditions

Design Note: The UI design for HomeSalad emphasizes the connection between player and environment, using natural metaphors and organic design elements to reinforce the game's themes of sustainability and community. The interface aims to be both functional and educational, helping players understand complex systems while maintaining an intuitive and accessible experience.

Technical Implementation

The user interface design described in this document is implemented using a robust technical architecture that integrates with Unity's Model Context Protocol (MCP) server. This implementation provides the foundation for the UI's functionality, responsiveness, and multiplayer capabilities.

Implementation Highlights

  • Layered Architecture: The UI is built on a layered architecture with clear separation of concerns between canvas management, event handling, UI state management, and panel implementation
  • MCP Server Integration: Integration with Unity's MCP server enables seamless multiplayer functionality and state synchronization
  • Event-Driven Communication: An event-driven architecture allows for decoupled communication between UI components and game systems
  • uMMORPG Component Refactoring: The UI system builds upon and extends the uMMORPG Remastered framework, customized for HomeSalad's specific needs

For detailed information about the technical implementation of the UI system, including architecture diagrams, component descriptions, and integration details, please refer to the UI-MCP Implementation documentation in the Technical Guide.

Implementation Note: When designing new UI features, consult both this design document and the technical implementation guide to ensure that your designs align with the established architecture and technical capabilities of the system.
