HomeSalad Technical Implementation Guide

This Technical Implementation Guide provides detailed specifications for implementing HomeSalad using the uMMORPG Remastered framework, including system architecture, core systems refactoring, specific system implementations, and multiplayer synchronization solutions.

Introduction

HomeSalad will be built on the uMMORPG Remastered framework, which provides a solid foundation for multiplayer RPG development in Unity. This Technical Implementation Guide details how to extend and modify uMMORPG to support HomeSalad's unique gameplay systems while maintaining the robust networking and persistence capabilities of the base framework.

The guide is intended for developers implementing HomeSalad, providing code examples, architectural diagrams, and detailed explanations of technical approaches to the game's key systems.

uMMORPG Remastered Integration

uMMORPG Remastered provides several key capabilities that make it suitable as a foundation for HomeSalad:

Strengths

  • Networking Framework: Mirror-based networking with client-server architecture
  • Character Systems: Player and NPC base classes with movement, stats, and interactions
  • Inventory System: Item management and equipment
  • Quest Framework: Basic quest tracking and completion
  • Database Integration: SQLite-based persistence
  • UI Framework: Core UI components for MMO interfaces

Limitations

  • Environmental Systems: No built-in support for seasons, weather, or environmental changes
  • Building Mechanics: No construction or placement systems
  • Gardening Systems: No plant growth or cultivation mechanics
  • Community Systems: Limited social and faction mechanics
  • Time System: Basic time without seasons or meaningful day/night cycles

The implementation approach will maintain uMMORPG's core architecture while extending it with HomeSalad-specific systems. This strategy minimizes risk while allowing for the addition of unique gameplay features.

Refactoring Strategy

The approach to extending uMMORPG Remastered follows these key principles:

  1. Minimal Core Modification: Avoid changing uMMORPG core files when possible to maintain compatibility with future updates.
  2. Extension Over Replacement: Extend existing classes rather than replacing them to preserve core functionality.
  3. Interface-Based Integration: Define clear interfaces between uMMORPG and HomeSalad systems to maintain separation of concerns.
  4. Addon Architecture: Use uMMORPG's addon system for non-intrusive extensions where possible.
  5. Namespace Isolation: Keep HomeSalad code in dedicated namespaces (HomeSalad.Core, HomeSalad.Gameplay, etc.) to prevent conflicts.

Implementation Approach

The implementation of HomeSalad will follow a phased approach, starting with core systems and progressively adding more complex gameplay features. Each phase builds upon the previous one, allowing for testing and validation at each step.

Key implementation considerations include:

  • Server Authority: The server remains authoritative for all gameplay systems to prevent cheating and ensure consistency.
  • Efficient Synchronization: Only significant changes are synchronized to clients to minimize network traffic.
  • Deterministic Calculations: Where possible, systems use deterministic calculations that can be predicted by clients between updates.
  • Database Schema Evolution: The database design allows for future expansion without breaking existing data.
  • Modular Design: Systems are designed with clear interfaces to allow for independent development and testing.

Code examples and detailed implementation instructions for each system are provided in the subsequent sections of this guide.
