Specific Systems

This section details the technical implementation of HomeSalad's specific gameplay systems, including gardening, building, environmental transformation, community relationships, and quest management.

Gardening System

The gardening system is a core gameplay mechanic that allows players to cultivate plants and manage growing spaces.

System Architecture

The gardening system is implemented through several interconnected components:

  • Plant: Core component for individual plants with growth stages, health, and resource needs
  • GardenBed: Container for plants with soil properties and grid-based planting
  • GardeningSystem: Global manager for plant databases and environmental effects
  • PlantVisualController: Handles visual representation of plants at different growth stages
  • SoilSystem: Manages soil properties, moisture, and nutrients

Key Technical Features

  • Growth Simulation: Time-based growth system with multiple stages and visual transitions
  • Environmental Factors: Plants respond to soil conditions, weather, seasons, and nearby plants
  • Grid-Based Planting: Spatial management system for organizing plants in garden beds
  • Procedural Variation: Plants have slight variations in appearance and growth patterns
  • Network Synchronization: Plant states are synchronized across clients using SyncVars

Integration Points

The gardening system integrates with several other systems:

  • Time System: Plant growth advances based on game time
  • Weather System: Weather conditions affect watering needs and growth rates
  • Inventory System: Seeds are consumed from inventory, harvests are added to inventory
  • Environmental System: Plants contribute to environmental metrics and transformation
  • Skill System: Player gardening skills affect success rates and yields

Building System

The building system allows players to construct structures and infrastructure in the game world.

System Architecture

The building system is implemented through these key components:

  • Building: Core component for structures with construction progress and functionality
  • BuildingSystem: Global manager for building placement and construction
  • BuildingFunction: Interface for functional components that can be attached to buildings
  • BuildingPlacementValidator: Validates building placement based on terrain and collision
  • BuildingVisualController: Manages visual representation during construction and customization

Key Technical Features

  • Modular Construction: Buildings are assembled from modular components
  • Placement System: Grid-based placement with collision and terrain validation
  • Construction Process: Multi-stage building process requiring resources and time
  • Functional Components: Buildings have pluggable functional components
  • Customization: Visual and functional customization options

Integration Points

The building system integrates with several other systems:

  • Resource System: Buildings require resources for construction and maintenance
  • Grid System: Buildings occupy space on a world grid
  • Environmental System: Buildings affect and are affected by environmental conditions
  • Power System: Buildings can produce, consume, and distribute power
  • Water System: Buildings can collect, store, and distribute water

Environmental Transformation System

The environmental transformation system manages the visual and functional changes to the environment based on player actions.

System Architecture

The environmental system is implemented through these components:

  • EnvironmentalRegion: Manages environmental state for a specific area
  • EnvironmentalSystem: Global manager for environmental regions and effects
  • EnvironmentalVisualController: Handles visual representation of environmental states
  • EnvironmentalAction: Represents player actions that affect the environment
  • EnvironmentalEffect: Ongoing effects that modify environmental metrics over time

Key Technical Features

  • Region-Based Management: Environment is divided into manageable regions
  • Multi-Metric System: Environmental health is measured across multiple metrics
  • Visual Transformation: Progressive visual changes based on environmental state
  • Action Propagation: Environmental actions have radius-based effects
  • Temporal Dynamics: Environmental changes occur over time with momentum

Integration Points

The environmental system integrates with several other systems:

  • Gardening System: Plants contribute to environmental metrics
  • Building System: Buildings can have positive or negative environmental impacts
  • Weather System: Weather affects and is affected by environmental conditions
  • Wildlife System: Animal presence is tied to environmental health
  • Quest System: Environmental transformation can trigger or complete quests

Community Relationship System

The community relationship system manages NPC relationships, faction standings, and social dynamics.

System Architecture

The relationship system is implemented through these components:

  • NPCRelationship: Manages individual NPC relationship dimensions and history
  • FactionReputation: Tracks player standing with factions and organizations
  • CommunitySystem: Global manager for relationships and reputation
  • RelationshipEvent: Records interactions that affect relationships
  • CommunityEvent: Represents events that affect multiple relationships

Key Technical Features

  • Multi-Dimensional Relationships: NPCs have trust, respect, and affinity dimensions
  • Relationship Memory: System tracks interaction history and context
  • Faction Reputation Levels: Progressive reputation tiers with unlockable benefits
  • Relationship Networks: NPCs have relationships with each other that affect behavior
  • Community Events: Events can affect multiple relationships simultaneously

Integration Points

The relationship system integrates with several other systems:

  • Dialogue System: Relationship affects available dialogue options
  • Quest System: Relationship levels unlock quests and affect outcomes
  • Trading System: Relationship affects prices and available items
  • Help System: NPCs provide different assistance based on relationship
  • Event System: Community events are influenced by relationship networks

Quest System

The quest system manages mission tracking, objectives, and rewards.

System Architecture

The quest system is implemented through these components:

  • Quest: Core component for individual quest instances with objectives and state
  • QuestObjective: Tracks progress for specific quest goals
  • QuestSystem: Global manager for quest tracking and availability
  • QuestTrigger: World objects that can start or advance quests
  • QuestReward: Defines rewards for completing quests

Key Technical Features

  • Objective Tracking: Multiple objective types with progress tracking
  • Branching Quests: Support for player choices affecting quest progression
  • Quest Dependencies: Quests can require completion of other quests
  • Dynamic Availability: Quests appear based on player actions and world state
  • Reward System: Multiple reward types including items, reputation, and environmental changes

Integration Points

The quest system integrates with several other systems:

  • Dialogue System: Quests can be started and advanced through dialogue
  • Inventory System: Quest items and rewards interact with inventory
  • Relationship System: Quests affect and are affected by NPC relationships
  • Environmental System: Quests can trigger environmental changes
  • Time System: Some quests have time constraints or time-based triggers

Technical Note: All specific systems are designed with modularity in mind, allowing for independent testing and iterative development. Systems communicate primarily through events and interfaces to maintain loose coupling.
