HomeSalad GameObject Integration

This page details each GameObject in HomeSalad, explaining how uMMORPG game assets, GameObjects, and data are being reused and extended. It includes code examples, integration explanations, and implementation progress tracking.

GameObject Overview

HomeSalad extends the uMMORPG Remastered framework by adding new GameObjects and components to support gardening, environmental transformation, building, and community systems. This page documents how these GameObjects are integrated with the base framework and how they interact with each other.

Player GameObjects

HomeSaladPlayer

The HomeSaladPlayer GameObject extends the base Player class from uMMORPG to add gardening, building, and community interaction capabilities.

Components:

HomeSaladPlayer.cs
Main player script that extends the uMMORPG Player class with HomeSalad-specific functionality.
GardeningSkill.cs
Manages the player's gardening abilities, including planting, watering, and harvesting.
BuildingSkill.cs
Handles the player's building capabilities, including placement and construction.
// Example of HomeSaladPlayer.cs extending uMMORPG Player
public class HomeSaladPlayer : Player
{
    // HomeSalad-specific player attributes
    [SyncVar] public int gardeningSkill;
    [SyncVar] public int buildingSkill;
    [SyncVar] public int communitySkill;
    
    // References to HomeSalad components
    public GardeningSkill gardeningComponent;
    public BuildingSkill buildingComponent;
    public CommunityInteraction communityComponent;
    
    // Command to plant a seed at the specified position
    [Command]
    public void CmdPlantSeed(string seedName, Vector3 position)
    {
        // Validate the player can plant here
        if (gardeningComponent.CanPlant(position))
        {
            // Plant the seed
            gardeningComponent.PlantSeed(seedName, position);
        }
    }
}
Implementation: 85% Complete

NPC GameObjects

HomeSaladNPC

The HomeSaladNPC GameObject extends the base NPC class from uMMORPG to add community roles, schedules, and relationship tracking.

Components:

HomeSaladNPC.cs
Main NPC script that extends the uMMORPG NPC class with HomeSalad-specific functionality.
NPCSchedule.cs
Manages the NPC's daily schedule and activities.
RelationshipTracker.cs
Tracks relationships with players and other NPCs.
Implementation: 75% Complete

Environment GameObjects

EnvironmentArea

The EnvironmentArea GameObject represents a specific area in the game world with its own environmental state, biodiversity, and transformation capabilities.

Components:

EnvironmentState.cs
Tracks the environmental state and properties of the area.
EnvironmentTransformation.cs
Handles visual and functional transformation of the environment based on its state.
Implementation: 70% Complete

Plant GameObjects

Plant

The Plant GameObject represents a growable plant in the game world, with growth stages, watering needs, and harvestable products.

Components:

Plant.cs
Main plant script that manages growth, health, and harvesting.
GrowthStages.cs
Handles visual representation of different growth stages.
Implementation: 80% Complete

Unity Console Integration

The HomeSalad documentation includes real-time integration with the Unity Console, allowing developers to view errors, warnings, and logs directly within the documentation.

[INFO] HomeSalad GameObject system initialized
[INFO] Player prefab loaded with HomeSalad components
[WARNING] NPC schedule system needs optimization
[ERROR] Missing reference to PlantGrowthManager in GardenPlot prefab
[INFO] Environment transformation system ready

This console output helps developers identify and resolve issues quickly during implementation.
